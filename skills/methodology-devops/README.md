## 核心能力

### 1. CI/CD管道设计 (8阶段)
- **计划** → **开发** → **构建** → **测试** → **发布** → **部署** → **运营** → **监控**
- 支持GitFlow和Trunk-based分支策略
- 自动化测试金字塔 (单元/集成/E2E)
- 多种部署策略 (蓝绿/金丝雀/滚动)

### 2. 基础设施即代码 (IaC)
- Terraform, CloudFormation, Ansible
- 模块化和环境隔离
- 状态管理和版本控制
- 密钥管理和安全实践

### 3. 监控和可观测性
- **Metrics** (Prometheus指标)
- **Logs** (ELK日志聚合)
- **Traces** (Jaeger分布式追踪)
- 告警策略和升级机制

### 4. 事件响应和SRE
- 事件分级 (P0/P1/P2/P3)
- 响应流程 (识别→响应→诊断→缓解→恢复→事后分析)
- Error Budget和Toil管理
- On-call轮值和Runbook

### 5. 配置管理
- 外部化配置和环境变量
- 配置中心 (Consul, etcd)
- 密钥管理 (Vault)
- 功能开关

### 6. DevOps文化
- CALMS框架 (Culture, Automation, Lean, Measurement, Sharing)
- 无责归咎文化
- Post-Mortem实践
- 持续改进

## DevOps流程

```
计划 → 开发 → 构建 → 测试 → 发布 → 部署 → 运营 → 监控
 ↑                                                    ↓
 └────────────────── 反馈和改进 ←─────────────────────┘
```

## 部署策略对比

| 策略 | 停机时间 | 回滚速度 | 资源成本 | 风险 | 适用场景 |
|------|---------|---------|---------|------|---------|
| 蓝绿部署 | 零 | 极快 | 高(2倍) | 低 | 关键业务,有状态服务 |
| 金丝雀发布 | 零 | 快 | 中 | 极低 | 高风险变更,渐进验证 |
| 滚动发布 | 零 | 中 | 低 | 中 | 无状态服务,标准更新 |

## DORA四大指标

| 指标 | Elite水平 | 目的 |
|------|----------|------|
| 部署频率 | 多次/天 | 衡量交付速度 |
| 变更前置时间 | < 1小时 | 衡量响应能力 |
| 平均恢复时间 (MTTR) | < 1小时 | 衡量稳定性 |
| 变更失败率 | 0-15% | 衡量质量 |

## 输出格式

标准化的中文DevOps实施指南:

1. **CI/CD管道设计** - 阶段/工具链/配置示例
2. **部署策略** - 选择理由/实施步骤/回滚计划
3. **监控方案** - 指标/告警/仪表盘
4. **事件响应计划** - 团队/流程/Runbook
5. **基础设施即代码** - 模块结构/部署流程
6. **改进建议** - 短期/中期/长期
7. **度量指标** - DORA指标追踪

## 质量标准

- [ ] CI管道每次提交自动触发
- [ ] 构建时间 < 10分钟
- [ ] 测试覆盖率 > 80%
- [ ] 部署全自动化 (测试环境)
- [ ] 回滚时间 < 5分钟
- [ ] 关键服务可用性 > 99.9%
- [ ] P0事件平均恢复时间 < 1小时
- [ ] 所有基础设施代码化

## DevOps成熟度

### Level 1: 初始级
- 手动部署、无自动化测试、反应式运维

### Level 2: 管理级
- 基本CI/CD、部分自动化、基础监控

### Level 3: 定义级
- 完整CI/CD、高测试覆盖率、全面监控

### Level 4: 量化级
- 基础设施即代码、多种部署策略、数据驱动

### Level 5: 优化级
- 全面自动化、AIOps实践、持续优化文化

## 适用场景

- ✅ 需要快速交付
- ✅ 频繁发布
- ✅ 微服务架构
- ✅ 云原生应用
- ✅ 需要高可用
- ✅ 追求自动化

## 不适用场景

- ❌ 嵌入式系统 (发布周期长)
- ❌ 严格审批流程 (无法自动化)
- ❌ 遗留系统 (难以改造)
- ❌ 小型项目 (工具链成本高)

## 常见挑战

❌ **文化阻力** → ✅ 建立联合团队,共享On-call
❌ **工具链复杂** → ✅ 整合工具,标准化流程
❌ **遗留系统** → ✅ 渐进式改造,Strangler Pattern

## 最佳实践

✅ 自动化一切可自动化的
✅ 小批量频繁发布
✅ 度量一切,数据驱动
✅ 快速反馈循环
✅ 基础设施即代码
✅ 无责归咎文化
✅ 持续学习和改进
