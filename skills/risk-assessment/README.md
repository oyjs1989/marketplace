## 核心能力

### 1. 风险识别 (5类风险)
- **技术风险**: 架构、技术选型、技术债务、性能
- **项目风险**: 进度、资源、依赖、沟通
- **业务风险**: 市场、竞争、合规、战略对齐
- **安全风险**: 数据泄露、攻击、漏洞、合规
- **运维风险**: 可用性、可维护性、灾难恢复

### 2. 风险分析
- 根因分析 (5 Whys)
- 影响分析 (进度/成本/质量)
- 触发条件识别
- 连锁反应分析

### 3. 风险量化
- **定性**: 高/中/低
- **半定量**: 1-5分或1-10分评分
- **定量**: 概率百分比、EMV、蒙特卡洛

### 4. 风险应对 (4T策略)
1. **Terminate (规避)**: 完全避免风险
2. **Treat (降低)**: 降低概率或影响
3. **Transfer (转移)**: 转移给第三方
4. **Tolerate (接受)**: 接受并准备应急

## 风险矩阵

可视化展示风险分布:

```
影响
  ↑
严重│  🔴🔴  🔴🔴  🔴🔴
    │
中等│  🟡🟡  🔴🔴  🔴🔴
    │
轻微│  🟢🟢  🟡🟡  🟡🟡
    │
    └─────────────────────→ 概率
      低     中     高

🔴 P0 高风险: 立即处理
🟡 P1 中风险: 密切监控
🟢 P2 低风险: 接受或延后
```

## 输出格式

标准化的中文风险评估报告:

1. **执行摘要** - 关键发现和总体风险水平
2. **风险清单** - P0/P1/P2分类详细列表
3. **风险分析** - 每个风险的深入分析
4. **风险矩阵** - 可视化风险分布
5. **缓解计划** - 按优先级的行动计划
6. **资源需求** - 成本和时间估算
7. **监控计划** - KRI和评审机制
8. **建议和结论** - 综合评估和决策点

## 风险评估方法

### 方法1: 风险矩阵
- **适用**: 快速定性评估
- **工具**: 概率×影响矩阵
- **输出**: 风险热力图

### 方法2: EMV (期望货币值)
- **适用**: 成本风险分析
- **公式**: EMV = 概率 × 影响金额
- **输出**: 定量风险值

### 方法3: 蒙特卡洛模拟
- **适用**: 复杂项目进度/成本风险
- **工具**: 随机模拟
- **输出**: 概率分布曲线

### 方法4: FMEA (失效模式分析)
- **适用**: 系统可靠性分析
- **公式**: RPN = 严重度 × 发生率 × 检测度
- **输出**: RPN优先级清单

## 应对策略选择

| 策略 | 适用场景 | 示例 |
|------|----------|------|
| 规避 | 不可接受的高风险 | 更换不成熟技术为成熟技术 |
| 降低 | 可以采取措施的风险 | 添加缓存降低数据库压力 |
| 转移 | 可转移给第三方 | 使用云服务转移基础设施风险 |
| 接受 | 低影响或应对成本高 | 接受罕见边界情况Bug |

## 风险监控

### 关键风险指标 (KRI)

**技术KRI**:
- 代码复杂度
- 技术债务比
- 测试覆盖率
- 错误率
- 响应时间

**项目KRI**:
- 进度偏差
- 资源利用率
- 需求变更率
- 团队流失率
- 缺陷趋势

**业务KRI**:
- 用户增长率
- 客户流失率
- NPS评分
- 市场份额
- 合规审计发现

### 评审频率

- **每日**: 检查关键指标,处理新风险
- **每周**: 风险清单评审,进度更新
- **每月**: 全面风险评估,趋势分析
- **每季度**: 战略风险评审,框架评估
