# 决策支持集成技能

这是一个综合性的决策支持技能,帮助在多个备选方案间做出理性、数据驱动的决策。可以独立使用或作为其他技能的集成器。

## 何时使用此技能

当用户请求以下任何内容时使用此技能:
- "帮我决策"
- "比较方案"
- "选哪个方法"
- "决策矩阵"
- "评估备选方案"
- "权衡分析"
- "技术选型"
- "方案对比"
- "如何选择"

## 核心能力

### 1. 多准则决策分析 (MCDA)
使用多准则决策分析方法,综合考虑多个评估维度:
- 层次分析法 (AHP - Analytic Hierarchy Process)
- 加权评分法 (Weighted Scoring)
- TOPSIS法 (逼近理想解)
- 决策矩阵

### 2. 方案比较
系统化比较多个备选方案:
- 定义评估标准和权重
- 量化各方案在每个标准上的表现
- 可视化比较结果
- 敏感度分析

### 3. 风险调整推荐
整合风险因素到决策中:
- 调用risk-assessment技能评估各方案风险
- 考虑风险容忍度
- 提供风险调整后的推荐

### 4. 经济可行性评估
整合成本效益分析:
- 调用cost-benefit-analysis技能
- 评估ROI和回收期
- 考虑资源约束

## 工作流程

### 第1步: 明确决策目标
- 理解需要决定什么
- 识别决策的背景和约束
- 明确决策的重要性和紧迫性
- 识别所有利益相关者

### 第2步: 识别备选方案
- 收集所有候选方案
- 确保方案覆盖范围合理
- 识别"不做任何改变"的基准方案
- 如果需要,可调用problem-solving技能生成方案

### 第3步: 定义评估标准
建立多维度评估标准:

**技术标准**:
- 技术可行性
- 技术成熟度
- 可扩展性
- 可维护性
- 性能表现

**业务标准**:
- 业务价值
- 上市时间
- 战略对齐
- 用户满意度

**经济标准**:
- 初期投入成本
- 运营成本
- 投资回报率(ROI)
- 回收期

**风险标准**:
- 技术风险
- 项目风险
- 业务风险
- 合规风险

**其他标准**:
- 团队技能匹配度
- 供应商稳定性
- 社区生态
- 未来趋势

### 第4步: 确定标准权重
根据决策上下文确定各标准的相对重要性:
- 使用AHP两两比较确定权重
- 或直接分配权重(总和100%)
- 考虑利益相关者的优先级

**权重分配原则**:
- 关键成功因素给予更高权重
- 考虑组织战略目标
- 平衡短期和长期利益

### 第5步: 评分和分析
对每个方案在每个标准上评分:
- 使用1-5分或1-10分评分
- 计算加权总分
- 排序所有方案
- 识别最优方案

### 第6步: 整合其他分析
**可选: 调用风险评估**
```markdown
如果决策涉及显著风险,调用 risk-assessment 技能:
- 评估每个方案的风险
- 量化风险影响
- 考虑风险调整后的得分
```

**可选: 调用成本效益分析**
```markdown
如果经济因素关键,调用 cost-benefit-analysis 技能:
- 详细计算每个方案的成本和效益
- 计算ROI、NPV、回收期
- 进行情景分析
```

### 第7步: 敏感度分析
分析决策的稳健性:
- 如果权重改变,结果如何变化?
- 如果评分有误差,结果如何变化?
- 识别关键影响因素

### 第8步: 综合推荐
提供清晰的决策建议:
- 推荐首选方案
- 说明推荐理由
- 识别关键考虑因素
- 提供实施建议
- 说明备选方案

## 决策方法

### AHP (层次分析法)
当标准复杂,难以直接确定权重时使用:

**步骤**:
1. 建立层次结构(目标-准则-方案)
2. 两两比较确定相对重要性
3. 计算权重
4. 一致性检验

**适用场景**: 战略决策、复杂多标准决策

### 加权评分法
最常用的方法,直观易懂:

**步骤**:
1. 定义评估标准
2. 分配权重
3. 对每个方案评分
4. 计算加权总分
5. 排序

**适用场景**: 大多数技术决策、方案选型

### TOPSIS法
逼近理想解,适合多方案多标准:

**步骤**:
1. 规范化决策矩阵
2. 确定理想解和负理想解
3. 计算各方案到理想解的距离
4. 计算相对贴近度
5. 排序

**适用场景**: 技术选型、供应商选择

## 决策陷阱

常见决策陷阱及避免方法:

❌ **锚定偏差**: 过度依赖第一印象
✅ **避免**: 独立评估每个方案

❌ **确认偏差**: 只寻找支持预设结论的证据
✅ **避免**: 主动寻找反对证据

❌ **群体思维**: 团队一致性压力导致不质疑
✅ **避免**: 鼓励多元观点和批判性思考

❌ **沉没成本谬误**: 因已投入而继续错误方向
✅ **避免**: 只考虑未来成本和效益

❌ **过度自信**: 高估成功概率
✅ **避免**: 参考历史数据和专家意见
