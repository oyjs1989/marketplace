# 成本效益分析技能 (Cost-Benefit Analysis Skill)

**版本**: 1.0.0
**语言**: 中文 (Chinese)

系统化的成本效益分析(CBA)技能,帮助量化和评估软件开发与IT投资的经济可行性,支持数据驱动的决策。

### 示例使用

**场景1: 新项目经济评估**
```
用户: "评估云迁移项目的成本效益"

系统: 执行全面CBA
1. 成本识别 (开发/采购/迁移/培训/运营/维护)
2. 效益量化 (成本节约/收入增加/效率提升)
3. 计算经济指标 (ROI/回收期/NPV)
4. 情景分析 (最佳/最可能/最差)
5. 敏感度分析 (识别关键因素)
6. 风险调整
7. 综合建议

输出: 完整的中文CBA报告 + 实施建议
```

**场景2: 方案对比**
```

2. 对比ROI、回收期、NPV

4. 敏感度分析
5. 推荐最优方案

输出: 对比分析报告 + 决策建议
```

## 核心能力

### 1. 成本识别 (6类成本)
- **一次性成本**: 开发、采购、迁移、培训
- **持续成本**: 运营、维护、许可证
- **隐性成本**: 培训、机会成本、风险成本
- **TCO**: 总拥有成本 (全生命周期)

### 2. 效益量化
- **有形效益**: 成本节约、收入增加、效率提升
- **无形效益**: 用户满意度、品牌价值、竞争优势

### 3. 经济指标 (5个)
- **ROI**: 投资回报率
- **回收期**: 收回投资所需时间
- **NPV**: 净现值 (考虑时间价值)
- **IRR**: 内部收益率
- **成本效益比**: 效益/成本

### 4. 情景分析
- 最佳情景 (乐观)
- 最可能情景 (现实)
- 最差情景 (悲观)
- 概率加权

### 5. 敏感度分析
- 单因素敏感度
- 双因素矩阵
- 盈亏平衡分析
- 关键影响因素识别

标准化的中文CBA报告:

1. **执行摘要** - ROI/回收期/NPV/建议
2. **成本分析** - 详细成本分类和TCO
3. **效益分析** - 有形和无形效益量化
4. **经济指标** - 5个核心指标计算

7. **风险调整** - 风险调整后指标
8. **对比分析** - 多方案对比(如有)
9. **建议和结论** - 实施建议

## 经济指标标准

### ROI阈值
```
< 0%: 亏损,拒绝
0-20%: 低回报,谨慎
20-50%: 中等,可考虑
50-100%: 好,值得投资
> 100%: 优秀,优先投资
```

### 回收期标准
```
< 2年: 优秀,快速回收
2-3年: 良好,可接受
3-5年: 一般,需评估
> 5年: 较长,高风险
```

### NPV决策
```
NPV > 0: 投资可行
NPV = 0: 盈亏平衡
NPV < 0: 投资不可行
```

## 效益量化方法

### 时间节约
```
效益 = 节省时间 × 人均成本 × 影响人数 × 频率

示例:
节省: 2小时/人/周
人均成本: ¥187.5/小时
影响人数: 50人
年度效益: 2 × 187.5 × 50 × 52 = ¥975,000
```

### 错误减少
```
效益 = 错误减少量 × 单次错误成本

示例:
减少: 80次/月
单次成本: ¥1,000
年度效益: 80 × 1,000 × 12 = ¥960,000
```

### 客户留存
```
效益 = 留存增加 × 客户LTV

示例:
留存增加: 200客户
LTV: ¥50,000
效益: 200 × 50,000 = ¥10,000,000
```

## 情景分析矩阵

| 情景 | 成本假设 | 效益假设 | 概率 | ROI |
|------|---------|---------|------|-----|
| 最佳 | -10% | +20% | 20% | 313% |
| 最可能 | 符合预算 | 80%实现 | 60% | 136% |
| 最差 | +20% | 60%实现 | 20% | 49% |

**期望ROI** = 20%×313% + 60%×136% + 20%×49% = 154%
