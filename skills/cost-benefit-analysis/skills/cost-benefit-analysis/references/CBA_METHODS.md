## CBA概述

成本效益分析(Cost-Benefit Analysis, CBA)是评估项目或决策经济可行性的系统化方法,通过比较所有成本和效益来确定是否值得投资。

### 核心原则

1. **全面性**: 识别所有相关成本和效益
2. **货币化**: 尽可能将成本和效益转换为货币价值
3. **时间价值**: 考虑货币的时间价值(折现)
4. **客观性**: 基于数据而非主观判断
5. **透明性**: 明确假设和计算方法

### 成本估算方法

#### 方法1: 类比估算
```
基于类似项目的历史数据

步骤:
1. 识别类似项目
2. 调整规模差异
3. 应用调整因子

示例:
类似项目成本: ¥1,000,000
规模比率: 1.5x
复杂度调整: 1.2x
估算成本: 1,000,000 × 1.5 × 1.2 = ¥1,800,000
```

#### 方法2: 参数估算
```
基于单位成本参数

示例 - 开发成本:
功能点数: 500
单位成本: ¥3,000/功能点
估算成本: 500 × 3,000 = ¥1,500,000
```

#### 方法3: 自下而上估算
```
详细分解后汇总

步骤:
1. WBS分解
2. 估算每个任务
3. 汇总总成本

最准确但最耗时
```

#### 方法4: 三点估算
```
考虑不确定性

成本 = (最乐观 + 4×最可能 + 最悲观) / 6

示例:
乐观: ¥800,000
最可能: ¥1,000,000
悲观: ¥1,400,000
期望成本: (800,000 + 4×1,000,000 + 1,400,000) / 6
         = ¥1,033,333
```

### 效益量化方法

#### 时间节约量化
```
步骤1: 识别受影响的活动
步骤2: 估算每次节约时间
步骤3: 估算频率
步骤4: 计算人力成本
步骤5: 汇总年度效益

示例:
活动: 数据录入
当前时间: 30分钟/次
改进后: 5分钟/次
节约: 25分钟/次
频率: 100次/天
工作日: 250天/年
人均成本: ¥200,000/年 → ¥125/小时

年度效益 = (25/60) × 100 × 250 × 125 = ¥1,302,083
```

#### 错误减少量化
```
步骤1: 确定当前错误率
步骤2: 估算改进后错误率
步骤3: 计算每次错误成本
步骤4: 汇总年度效益

示例:
当前错误: 500次/年
改进后: 100次/年
减少: 400次/年
每次成本: ¥2,000 (人力 + 客户影响)

年度效益 = 400 × 2,000 = ¥800,000
```

#### 客户价值量化
```
LTV (客户生命周期价值) = 年均收入 × 平均留存年限 × 利润率

示例:
年均收入: ¥10,000
留存年限: 5年
利润率: 30%
LTV = 10,000 × 5 × 0.3 = ¥15,000

改进:
留存率从80%升至85%
客户基数: 10,000
留存增加: 10,000 × 5% = 500客户
效益: 500 × 15,000 = ¥7,500,000
```

## 经济指标计算

### 1. ROI (投资回报率)

#### 公式
```
ROI = (总效益 - 总成本) / 总成本 × 100%
```

#### 计算步骤
```markdown
1. [ ] 汇总所有成本
2. [ ] 汇总所有效益
3. [ ] 计算净效益 = 效益 - 成本
4. [ ] 计算ROI
5. [ ] 与基准比较
```

#### 示例
```
总成本: ¥2,000,000
总效益: ¥4,200,000
净效益: ¥2,200,000
ROI = 2,200,000 / 2,000,000 × 100% = 110%

解读: 每投入¥1,获得¥2.1回报
```

#### ROI阈值
```
< 0%: 亏损,拒绝
0-20%: 低回报,谨慎
20-50%: 中等,可考虑
50-100%: 好,值得投资
> 100%: 优秀,优先投资
```

### 2. 回收期 (Payback Period)

#### 公式
```
简单回收期 = 初期投资 / 年均净现金流

或逐年累积直至转正
```

#### 计算示例
```
年度   成本      效益     净现金流  累积
0    -1,000,000      0  -1,000,000  -1,000,000
1      -200,000  600,000    400,000    -600,000
2      -200,000  800,000    600,000         0 ✓

回收期 = 1 + 600,000/600,000 = 2年
```

#### 决策标准
```
< 2年: 优秀,快速回收
2-3年: 良好,可接受
3-5年: 一般,需评估
> 5年: 较长,高风险
```

### 3. NPV (净现值)

#### 公式
```
NPV = Σ [CFt / (1+r)^t]

其中:
CF_t = 第t年净现金流
r = 折现率
t = 年份
```

#### 折现率选择
```
WACC (加权平均资本成本)
或
公司要求收益率
或
行业基准收益率

常用范围: 8-15%
```

#### 计算示例
```
折现率: 10%
年度  净现金流    折现系数    现值
0    -1,000,000   1.0000    -1,000,000
1       400,000   0.9091       363,636
2       600,000   0.8264       495,868
3       800,000   0.7513       601,036
4       900,000   0.6830       614,738
5     1,000,000   0.6209       620,921

NPV = 1,696,199
```

#### 决策标准
```
NPV > 0: 投资可行
NPV = 0: 盈亏平衡
NPV < 0: 投资不可行

多方案比较: 选NPV最大的
```

### 4. IRR (内部收益率)

#### 定义
```
使NPV=0的折现率

即: Σ [CFt / (1+IRR)^t] = 0
```

#### 计算方法
```
使用Excel IRR函数或迭代计算
```

#### 决策标准
```
IRR > WACC: 可行
IRR = WACC: 临界
IRR < WACC: 不可行

比较: 选IRR最高的
```

### 5. 成本效益比

#### 公式
```
成本效益比 = 总效益现值 / 总成本现值
```

#### 决策标准
```
> 2.0: 优秀
1.5-2.0: 良好
1.0-1.5: 可接受
< 1.0: 不可行
```

## 情景分析方法

### 三情景模型

#### 最佳情景 (Optimistic)
```markdown
假设:
- [ ] 所有事情顺利进行
- [ ] 成本低于预算10-20%
- [ ] 效益高于预期20-30%
- [ ] 风险未发生

用途: 了解上限潜力
```

#### 最可能情景 (Realistic)
```markdown
假设:
- [ ] 正常执行
- [ ] 成本符合预算±5%
- [ ] 效益实现80-90%
- [ ] 部分小风险发生

用途: 基准决策依据
```

#### 最差情景 (Pessimistic)
```markdown
假设:
- [ ] 遇到困难和延误
- [ ] 成本超支20-30%
- [ ] 效益仅实现50-60%
- [ ] 主要风险发生

用途: 评估下行风险
```

### 概率加权
```
期望值 = Σ (情景值 × 概率)

示例:
最佳(20%): ROI 200%
最可能(60%): ROI 100%
最差(20%): ROI 20%

期望ROI = 0.2×200% + 0.6×100% + 0.2×20%
        = 40% + 60% + 4%
        = 104%
```

## 敏感度分析方法

### 单因素敏感度

#### 步骤
```markdown
1. [ ] 选择关键变量
2. [ ] 确定变化范围(±10%, ±20%)
3. [ ] 逐个改变变量值
4. [ ] 重新计算指标
5. [ ] 绘制敏感度图
6. [ ] 识别敏感因素
```

#### 敏感度计算
```
敏感度系数 = (指标变化% / 变量变化%)

示例:
变量"采用率"增加10%
ROI从100%升至130%
敏感度 = 30% / 10% = 3.0

高度敏感!
```

### 双因素敏感度

#### 矩阵分析
```
创建二维矩阵,同时变化两个变量

示例: 成本 vs 采用率

            采用率
          60%   80%   100%
成本  80万  50%   80%   110%
     100万  30%   60%    90%
     120万  10%   40%    70%
```

### 盈亏平衡分析

#### 目的
识别关键变量需要达到的临界值

#### 方法
```markdown
1. [ ] 设定目标指标(如ROI=0)
2. [ ] 逆向计算变量值
3. [ ] 评估可行性
```

#### 示例
```
问题: 采用率至少多少才能盈亏平衡?

设: ROI = 0
则: 总效益 = 总成本
    采用率 × 单用户效益 × 用户数 = 总成本
    采用率 = 总成本 / (单用户效益 × 用户数)
    采用率 = 2,000,000 / (10,000 × 50)
    采用率 = 40%

结论: 至少40%采用率才能盈亏平衡
```

## 最佳实践

### CBA执行清单

```markdown
准备阶段:
- [ ] 明确决策或投资内容
- [ ] 定义分析范围和时间框架
- [ ] 识别利益相关者
- [ ] 收集历史数据和基准

分析阶段:
- [ ] 全面识别成本和效益
- [ ] 量化所有可量化项
- [ ] 记录无法量化的项
- [ ] 明确所有假设
- [ ] 计算经济指标
- [ ] 进行情景和敏感度分析

验证阶段:
- [ ] 检查计算正确性
- [ ] 验证假设合理性
- [ ] 进行同行评审
- [ ] 测试关键假设

报告阶段:
- [ ] 编写清晰报告
- [ ] 可视化关键结果
- [ ] 明确建议和理由
- [ ] 识别关键风险和假设
```

### 常见陷阱

❌ **遗漏隐性成本**: 培训、变革管理、机会成本
❌ **高估效益**: 过于乐观的假设
❌ **忽略时间价值**: 未使用折现
❌ **分析麻痹**: 过度分析延误决策
❌ **确认偏差**: 只寻找支持结论的数据
❌ **沉没成本**: 考虑已发生的不可收回成本

### 成功要素

✅ 全面识别成本和效益
✅ 基于数据而非猜测
✅ 考虑货币时间价值
✅ 进行多情景分析
✅ 识别关键敏感因素
✅ 明确记录假设
✅ 持续监控和更新
