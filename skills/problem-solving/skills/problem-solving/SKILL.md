# 问题解决编排器

这是一个综合性的问题解决技能,通过协调5个专业认知代理来提供多视角、系统化的问题分析和解决方案。

## 何时使用此技能

当用户请求以下任何内容时使用此技能:
- "解决这个问题"
- "帮我分析"
- "系统性思考"
- "问题分析"
- "头脑风暴"
- "设计思维"
- "全局视角"
- "整体分析"
- "架构设计"
- "技术方案"

## 架构设计

此技能使用**编排器模式**,协调5个专业认知代理并行工作:

| 代理 | 颜色 | 核心职责 |
|------|------|---------|
| systems-thinking | 🔵 蓝色 | 全局视角、涌现属性、杠杆点识别 |
| modeling-abstraction | 🟢 绿色 | 概念模型、抽象层次、模式提取 |
| decomposition | 🟣 紫色 | 层次分解、组件隔离、方案合成 |
| iteration | 🟠 橙色 | 增量改进、反馈循环、渐进增强 |
| pattern-recognition | 🔷 青色 | 模式发现/重构/集成/促进/需求确定 |

## 代理选择逻辑

### 默认行为
对于综合性问题分析,**默认激活所有5个代理**以确保全面覆盖。

### 智能选择
基于问题类型和用户关键词,可以选择性激活相关代理:

**基于关键词**:
- "全局"、"整体"、"系统边界" → systems-thinking
- "模型"、"抽象"、"设计"、"概念" → modeling-abstraction
- "分解"、"组件"、"拆分"、"模块化" → decomposition
- "迭代"、"增量"、"反馈"、"渐进" → iteration
- "模式"、"类似问题"、"重构"、"最佳实践" → pattern-recognition

**基于问题类型**:
- **架构设计问题** → 全部5个代理
- **技术债务分析** → systems-thinking + decomposition + pattern-recognition
- **新功能开发** → modeling-abstraction + decomposition + iteration
- **性能优化问题** → systems-thinking + pattern-recognition
- **复杂Bug调查** → systems-thinking + decomposition + pattern-recognition

## 工作流程

### 第1步: 问题理解
- 阅读并理解用户描述的问题
- 识别问题的关键词和类型
- 确定需要激活哪些代理

### 第2步: 代理协调
使用Task工具并行调用选定的代理:

```markdown
对于全面分析,并行调用所有5个代理:
- Task: systems-thinking代理 (蓝色)
- Task: modeling-abstraction代理 (绿色)
- Task: decomposition代理 (紫色)
- Task: iteration代理 (橙色)
- Task: pattern-recognition代理 (青色)
```

每个代理在其对应的 `agents/*.md` 文件中定义,包含:
- 专注领域和职责
- 分析框架和方法
- 输出格式

### 第3步: 综合分析
整合所有代理的输出,形成统一的问题解决方案:
- 识别不同视角的共同洞察
- 发现互补的解决策略
- 解决代理间的冲突或矛盾

### 第4步: 输出生成
按照标准化的中文格式输出完整分析报告。

## 语言要求

- ✅ **所有问题描述、建议、分析**: 必须使用中文
- ✅ **文件路径、代码片段**: 保持原文
- ✅ **技术术语**: 首次使用提供英文,后续使用中文 (如: "迭代 (Iteration)")

## 与其他技能的集成

此编排器技能可以与其他专业技能配合使用:

### 风险评估
当问题分析需要详细的风险评估时,可以调用 `risk-assessment` 技能:
- 识别技术风险、项目风险、业务风险
- 量化风险概率和影响
- 制定缓解策略

### 成本效益分析
当需要经济可行性评估时,可以调用 `cost-benefit-analysis` 技能:
- 量化成本和效益
- 计算ROI和回收期
- 情景分析

### 决策支持
当需要在多个方案间做决策时,可以调用 `decision-support` 技能:
- 多准则决策分析
- 决策矩阵
- 敏感度分析
