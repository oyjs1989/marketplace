# 问题解决编排器 (Problem-Solving Orchestrator)

**版本**: 1.0.0
**语言**: 中文 (Chinese)

综合性问题解决技能,通过协调5个专业认知代理提供多视角、系统化的问题分析和解决方案。

## 快速开始

### 触发关键词

当你需要系统化分析问题时,使用以下任何关键词:

- "解决这个问题"
- "帮我分析"
- "系统性思考"
- "问题分析"
- "头脑风暴"
- "设计思维"
- "架构设计"
- "全局视角"

### 示例使用

**场景1: 性能问题分析**
```
用户: "帮我分析一个性能问题:用户登录接口响应慢"

系统: 激活5个认知代理并行分析
- 🔵 系统化思维: 分析系统边界和依赖关系
- 🟢 建模与抽象: 构建认证流程模型
- 🟣 分解策略: 分解登录流程到各个组件
- 🟠 迭代方法: 规划分阶段优化计划
- 🔷 模式识别: 识别性能优化模式

输出: 完整的中文分析报告 + 可操作的优化方案
```

**场景2: 架构设计**
```
用户: "帮我设计一个微服务架构"

系统: 综合运用所有5个代理
- 系统化思维: 定义系统边界,识别服务依赖
- 建模与抽象: 设计领域模型和服务接口
- 分解策略: 划分微服务边界
- 迭代方法: 规划渐进式迁移策略
- 模式识别: 应用微服务设计模式

输出: 架构设计方案 + 实施路线图
```

## 架构设计

### 编排器模式

本技能采用**编排器 + 多代理**架构:

```
             主编排器 (SKILL.md)
                    |
        自动选择并协调代理
                    |
    ┌───────┬───────┼───────┬───────┐
    |       |       |       |       |
  🔵      🟢      🟣      🟠      🔷
systems  model  decomp  iter  pattern
-think   -ing   -ition  -ation  -recog
```

### 5个认知代理

| 代理 | 职责 | 文件 |
|------|------|------|
| 🔵 系统化思维 | 全局视角、涌现属性、杠杆点 | `agents/systems-thinking.md` |
| 🟢 建模与抽象 | 概念模型、抽象层次、模式提取 | `agents/modeling-abstraction.md` |
| 🟣 层次分解 | 层次分解、组件隔离、方案合成 | `agents/decomposition.md` |
| 🟠 迭代思维 | 增量改进、反馈循环、渐进增强 | `agents/iteration.md` |
| 🔷 模式识别 | 模式发现/重构/集成/促进/需求确定 | `agents/pattern-recognition.md` |
